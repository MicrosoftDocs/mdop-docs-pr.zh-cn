# 应用程序虚拟化术语表

</tr>

<tbody>
<tr class="odd">
<td align="left"><p>活动升级</p></td>
<td align="left"><p>允许将新版本的应用程序添加到应用程序 V 管理服务器或流服务器的升级，而不会影响当前运行该应用程序的用户。</p></td>
</tr>
<tr class="even">
<td align="left"><p>应用程序源根目录</p></td>
<td align="left"><p>对允许替代 HREF 元素的 OSD 基本代码（例如，源位置）的注册表项的引用。 此注册表值使管理员或 ESD 系统能够确保根据规划的拓扑管理方案执行应用程序加载。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Application Virtualization 驱动器</p></td>
<td align="left"><p>默认虚拟应用程序客户端驱动器（Q：）从其运行顺序应用程序。</p></td>
</tr>
<tr class="even">
<td align="left"><p>应用程序虚拟化报告</p></td>
<td align="left"><p>为数据分析收集的虚拟应用程序信息。 为自定义视图和解释的程序集收集数据。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>AutoLoad 配置参数</p></td>
<td align="left"><p>使虚拟化应用程序的辅助功能块能够在后台自动流式处理到客户端的客户端运行时策略配置参数。</p></td>
</tr>
<tr class="even">
<td align="left"><p>分支程序包</p></td>
<td align="left"><p>升级现有的顺序应用程序包，并与原始已排序的应用程序包并行运行。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>动态套件合成</p></td>
<td align="left"><p>允许虚拟应用程序包允许依赖插件或中间件程序包使用虚拟环境的应用程序虚拟化功能。 此功能支持插件和中间件程序包使用主程序包的注册表设置，并且程序包的行为和交互方式与在 comuter 上本地安装它们的方式相同。</p></td>
</tr>
<tr class="even">
<td align="left"><p>.ICO 文件</p></td>
<td align="left"><p>用于启动序列化应用程序的客户端桌面上的图标的文件。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>图标源根目录</p></td>
<td align="left"><p>对注册表项的引用，该注册表项允许在发布期间为顺序应用程序包指定源位置以进行图标检索。 图标源根目录仅支持 UNC 格式（而不支持 Url）。</p></td>
</tr>
<tr class="even">
<td align="left"><p>安装目录</p></td>
<td align="left"><p>应用程序虚拟化 sequencer 的安装程序用于放置其文件的目录。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization 桌面客户端</p></td>
<td align="left"><p>驻留在基于 Windows 的计算机桌面上的应用程序，它与 Microsoft System Center 虚拟应用程序服务器进行通信和身份验证以接收应用程序代码，并允许序列化的应用程序在本地运行。</p></td>
</tr>
<tr class="even">
<td align="left"><p>用于终端服务的 Microsoft Application 虚拟化</p></td>
<td align="left"><p>应用程序虚拟化功能，包括客户端和服务器组件，在终端服务环境中运行。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization 管理控制台</p></td>
<td align="left"><p>用于管理应用程序虚拟化管理系统的 Microsoft 管理控制台的集中式管理单元。</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization 管理系统</p></td>
<td align="left"><p>应用程序虚拟化平台的特定部署，其中包括由单个数据存储管理的所有组件。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization 管理 Web 服务</p></td>
<td align="left"><p>充当应用程序虚拟化管理控制台和数据存储之间的媒介的服务。 首先，它对其用户进行身份验证，然后允许授权的管理员操纵数据存储中的数据。</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization 平台</p></td>
<td align="left"><p>用于创建、存储、分发和运行虚拟化应用程序的 Microsoft 产品的名称。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization Sequencer</p></td>
<td align="left"><p>用于监视和记录应用程序的安装和设置过程的应用程序，以便可以在虚拟环境中对应用程序进行排序和运行。</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization 终端服务客户端</p></td>
<td align="left"><p>驻留在终端服务器上的应用程序，它与 Microsoft 虚拟应用程序服务器进行通信和身份验证以接收应用程序代码，并允许序列化的应用程序在本地运行。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft System Center 应用程序虚拟化管理服务器</p></td>
<td align="left"><p>两种应用程序虚拟化服务器类型中的一种，可用于对序列化应用程序包进行流式传输。 除了流虚拟应用程序包之外，应用程序虚拟化管理服务器还提供了其他服务，如发布、管理、报告等。</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft System Center 应用程序虚拟化流服务器</p></td>
<td align="left"><p>两种应用程序虚拟化服务器类型中的一种，可用于对序列化应用程序包进行流式传输。 流服务器仅将应用程序流处理到客户端计算机，并且不提供其他服务，如发布、管理、报告等。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>开放软件描述符（OSD）文件</p></td>
<td align="left"><p>基于 XML 的文件，指导客户了解如何从应用程序虚拟化管理/流服务器检索顺序化应用程序，以及如何在其虚拟环境中运行序列化的应用程序。</p></td>
</tr>
<tr class="even">
<td align="left"><p>OSD 源根目录</p></td>
<td align="left"><p>对注册表项的引用，该注册表项允许在发布期间为应用程序包提供对 OSD 文件检索的源位置的规范。 OSD 源根目录仅支持 UNC 格式（而不支持 Url）。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>程序包根目录</p></td>
<td align="left"><p>排序计算机上安装序列化应用程序包的文件的目录。 此目录还存在于将对序列化的应用程序进行流式处理的计算机上。</p></td>
</tr>
<tr class="even">
<td align="left"><p>主要功能块</p></td>
<td align="left"><p>运行应用程序所需的应用程序包中的最小内容。 主功能块中的内容在排序的应用阶段中标识，通常包含最常用的应用程序功能的内容。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>辅助功能块</p></td>
<td align="left"><p>不包含在主要功能块中的应用程序包的其余部分。 使用应用程序功能时，此内容将按需流式传输到客户端。</p></td>
</tr>
<tr class="even">
<td align="left"><p>顺序应用程序</p></td>
<td align="left"><p>由 Sequencer 监视的应用程序被分解为主功能块和辅助功能块，流出到运行 Microsoft Application Virtualization 终端服务客户端或 Microsoft Application Virtualization 桌面客户端的计算机，并可在其自己的虚拟环境内运行。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>顺序应用程序包</p></td>
<td align="left"><p>构成虚拟应用程序并允许虚拟应用程序运行的文件。 这些文件是在排序后创建的，具体包括 .osd、sprj、和 .ico 文件。</p></td>
</tr>
<tr class="even">
<td align="left"><p>引出</p></td>
<td align="left"><p>使用 Application Virtualization Sequencer 创建应用程序包的过程。 在此过程中，将监视应用程序，配置其快捷方式，并创建一个序列化的应用程序包，其中包含 .osd、sprj、和 .ico 文件。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>计算机排序</p></td>
<td align="left"><p>用于执行序列化和创建顺序应用程序包的计算机。</p></td>
</tr>
<tr class="even">
<td align="left"><p>序列化向导</p></td>
<td align="left"><p>分步向导，指导用户对应用程序进行排序，包括程序包配置、安装应用程序或要进行排序的应用程序包。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>SFT</p></td>
<td align="left"><p>一个文件，其中包含一个或多个排序器已打包到流块中的序列化应用程序以及关联的传递信息。 SFT 文件存储在必须将打包的应用程序流式传输到客户端的每台服务器上。</p></td>
</tr>
<tr class="even">
<td align="left"><p>SFTMIME</p></td>
<td align="left"><p>用于设置和管理应用程序、文件类型关联以及由应用程序虚拟化桌面和终端服务客户端管理的桌面配置服务器的命令行界面。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>SPRJ 文件</p></td>
<td align="left"><p>基于 XML 的 Sequencer 项目文件，其中 Sequencer 存储其排除项和分析项信息。 SPRJ 文件在创建应用程序记录和升级到程序包的过程中广泛使用。</p></td>
</tr>
<tr class="even">
<td align="left"><p>虚拟应用程序</p></td>
<td align="left"><p>由 Sequencer 打包以在自包含虚拟环境中运行的应用程序。 虚拟环境包含在客户端上运行应用程序所需的信息，而无需本地安装应用程序。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>虚拟 COM</p></td>
<td align="left"><p>管理由虚拟环境中运行的应用程序进程创建的 COM 对象并防止与在虚拟环境外创建的相同对象冲突的子系统。</p></td>
</tr>
<tr class="even">
<td align="left"><p>虚拟目录</p></td>
<td align="left"><p>一个不透明目录，其中只有在应用程序包中定义的文件和子目录或通过与虚拟环境中的应用程序交互而创建的文件和子目录才可见。 应用程序不会看到具有相同名称的本地目录中的任何文件。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>虚拟环境</p></td>
<td align="left"><p>定义可用于从顺序应用程序包启动的应用程序进程的资源的运行时容器。</p></td>
</tr>
<tr class="even">
<td align="left"><p>虚拟文件</p></td>
<td align="left"><p>虚拟环境中映射到备用目标位置的文件名。 虚拟文件将显示在包含目录中的其他文件旁边，无论该目录是虚拟目录还是本地目录。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>虚拟文件系统</p></td>
<td align="left"><p>从虚拟环境中运行的应用程序进程截获和重定向文件系统请求的子系统。 这些请求基于应用程序包中定义的虚拟文件和目录进行处理，并通过与虚拟应用程序进行交互来创建或修改。</p></td>
</tr>
<tr class="even">
<td align="left"><p>虚拟注册表</p></td>
<td align="left"><p>从虚拟环境中运行的应用程序进程截取和重定向注册表的注册表项请求的子系统。 重定向基于在应用程序包中定义的注册表信息，并通过与虚拟应用程序的交互创建或修改。</p></td>
</tr>
<tr class="odd">
<td align="left"><p>虚拟服务</p></td>
<td align="left"><p>充当虚拟环境中运行的服务的服务控制管理器（SCM）的子系统。</p></td>
</tr>
</tbody>
</table>
