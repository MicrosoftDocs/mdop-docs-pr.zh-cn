## <span data-ttu-id="5aa4f-123">有关 AGPM 4.0 的已知问题</span><span class="sxs-lookup"><span data-stu-id="5aa4f-123">Known issues with AGPM 4.0</span></span>


### <span data-ttu-id="5aa4f-124">"从生产中导入" 命令不会将设置导入已签出的 GPO 中</span><span class="sxs-lookup"><span data-stu-id="5aa4f-124">Import from Production command does not import settings into a GPO that is checked out</span></span>

<span data-ttu-id="5aa4f-125">如果在生产环境中编辑 GPO，则必须从生产导入 GPO 以更新脱机存档中的 GPO。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-125">If you edit a GPO in the production environment, you must import the GPO from production to update the GPO in the offline archive.</span></span> <span data-ttu-id="5aa4f-126">"**从生产中导入**" 命令旨在让你在完成编辑之前执行最终生产备份，以便你可以在需要时回滚到生产备份。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-126">The **Import from Production** command is intended to let you perform a final production backup before you finish editing so that you can roll back to the production backup if it is necessary.</span></span>

<span data-ttu-id="5aa4f-127">如果在运行 "**从生产中导入**" 命令时已签出 gpo，则不会将生产更改合并到 GPO 的已签出版本中。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-127">If the GPO is checked out when you run the **Import from Production** command, then the production changes are not incorporated into the checked out version of the GPO.</span></span> <span data-ttu-id="5aa4f-128">但是，即使该版本不可编辑，也会将该 GPO 的导入版本添加到 GPO 的历史记录中。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-128">However, the imported version of the GPO is added to the history of the GPO even though that version is not available to be edited.</span></span> <span data-ttu-id="5aa4f-129">GPO 签入后，该版本将取代存档中的导入版本，但这两个版本都在 GPO 的历史记录中可用。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-129">When the GPO is checked in, that version will supersede the imported version in the archive, but both are available in the history of the GPO.</span></span>

<span data-ttu-id="5aa4f-130">**解决方法：** 在从生产中导入之前，请确保已签入该 GPO。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-130">**Workaround:** Make sure that the GPO is checked in before you import it from production.</span></span> <span data-ttu-id="5aa4f-131">如果在导入该 GPO 之前未签入该 GPO，则可以使用 "**撤消签出**" 命令放弃所做的更改，然后回退到从生产导入的 gpo 版本。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-131">If the GPO was not checked in before you imported it, you can use the **Undo Check Out** command to discard your changes and roll back to the version of the GPO that you imported from production.</span></span>

### <span data-ttu-id="5aa4f-132">在使用多站点 Active Directory 拓扑的环境中，已签出的 Gpo 无法编辑几分钟</span><span class="sxs-lookup"><span data-stu-id="5aa4f-132">Checked out GPOs cannot be edited for several minutes in an environment that uses a multiple site Active Directory topology</span></span>

<span data-ttu-id="5aa4f-133">AGPM 使用客户端/服务器模型。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-133">AGPM uses a client/server model.</span></span> <span data-ttu-id="5aa4f-134">AGPM 服务器和 AGPM 客户端分别为组策略操作确定其自己最近的域控制器。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-134">The AGPM Server and the AGPM Client each determine their own closest domain controller for Group Policy operations.</span></span> <span data-ttu-id="5aa4f-135">使用 AGPM 客户端签出 GPO 时，它实际上是一个 AGPM 服务器，它将 GPO 从脱机存档检查到 SYSVOL 文件夹中的临时文件夹。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-135">When you check out a GPO by using an AGPM Client, it is actually the AGPM Server that checks the GPO out from the offline archive to a temporary folder in the SYSVOL folder.</span></span>

<span data-ttu-id="5aa4f-136">如果 AGPM 服务器和 AGPM 客户端位于不同的站点，则临时签出的 GPO 可能不会在本地站点的域控制器上出现几分钟或最多30分钟，因为 SYSVOL 复制延迟。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-136">If the AGPM Server and the AGPM Client are in different sites, then the temporary checked out GPO may not be present on the local site's domain controller for several minutes or up to 30 minutes due to SYSVOL replication latency.</span></span> <span data-ttu-id="5aa4f-137">在这种情况下，你无法在 AGPM 客户端上使用 GPMC 编辑已签出的 GPO，直到已签出 GPO 的 SYSVOL 复制发生。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-137">In this situation, you cannot edit the checked out GPO using the GPMC on an AGPM Client until SYSVOL replication of the checked out GPO has occurred.</span></span>

<span data-ttu-id="5aa4f-138">**解决方法：** 最佳做法是，你应将 AGPM 客户端放置在与它们连接的 AGPM 服务器相同的站点中，以便你无需等待 SYSVOL 复制发生，然后才能编辑已签出的 GPO。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-138">**Workaround:** As a best practice, you should position AGPM Clients in the same site as the AGPM Server to which they connect so that you do not have to wait for SYSVOL replication to occur before you can edit a checked out GPO.</span></span>

### <span data-ttu-id="5aa4f-139">如果你的帐户没有存档的权限，则 AGPM 无法读取备份限制</span><span class="sxs-lookup"><span data-stu-id="5aa4f-139">AGPM cannot read the backup limit if your account does not have permissions for the archive</span></span>

<span data-ttu-id="5aa4f-140">在 AGPM 客户端上，如果使用尚未委派给 AGPM 存档权限的帐户登录，请启动组策略管理控制台（GPMC），然后单击 "**更改控制**"，收到以下错误。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-140">On an AGPM Client, if you log on by using an account that has not been delegated permissions to the AGPM archive, start the Group Policy Management Console (GPMC), and then click **Change Control**, you receive the following error.</span></span>

``` syntax
Failed to read backup purge limit for this domain. 

The following error occurred: 
You do not have sufficient permissions to perform this operation. 
Microsoft.Agpm.AccessDeniedException (80070005)
```

<span data-ttu-id="5aa4f-141">**解决方法：** 与 AGPM 管理员（完全控制）联系，请求他们为您的帐户委派对 AGPM 的访问权限。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-141">**Workaround:** Contact an AGPM Administrator (Full Control) and request that they delegate access to AGPM for your account.</span></span> <span data-ttu-id="5aa4f-142">如果你是 AGPM 管理员，请使用分配了 AGPM 管理员角色的帐户登录，以便你可以为其他帐户委派访问权限。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-142">If you are an AGPM Administrator, log on by using an account to which the AGPM Administrator role is assigned so that you can delegate access for the additional account.</span></span> <span data-ttu-id="5aa4f-143">有关详细信息，请参阅 AGPM 帮助中的 "委派域级别对存档的访问权限"。</span><span class="sxs-lookup"><span data-stu-id="5aa4f-143">For more information, see "Delegate Domain-Level Access to the Archive" in the AGPM Help.</span></span>
